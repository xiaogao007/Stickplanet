## 🎯 核心设计理念

### 1. 用户体验至上
**原则说明**：用户使用体验是设计的最高优先级，所有设计决策都应围绕提升用户体验展开。

**具体要求**：
- **直观易用**：界面元素清晰，操作流程简单，降低学习成本
- **反馈及时**：所有用户操作都应有明确的视觉或触觉反馈
- **容错性强**：提供撤销、确认等机制，防止误操作
- **加载优化**：减少等待时间，优化加载体验（骨架屏、渐进式加载）
- **无障碍设计**：考虑色盲用户、文字大小可调、清晰的状态提示

### 2. 情绪价值优先
**原则说明**：产品不仅要实现功能，更要为用户带来情感上的满足和激励，增强用户粘性。

**具体要求**：
- **正向激励**：通过动画、文案、视觉反馈等方式庆祝用户成就
- **温暖友好**：使用温暖的问候语、鼓励性文案，营造友好氛围
- **成就感展示**：突出显示用户的数据成果（连续天数、总天数等）
- **个性化表达**：根据用户状态和时间段提供个性化的内容
- **社交认同**：通过分享、互动等方式满足用户的社交需求

### 3. 视觉美感与科技感
**原则说明**：页面设计要美观、现代、具有科技感，给用户带来视觉享受。

**具体要求**：
- **现代风格**：采用科技感设计语言，深色背景+霓虹色彩
- **层次丰富**：通过阴影、渐变、发光等效果营造空间层次
- **动效流畅**：使用流畅的动画和过渡效果，增强视觉吸引力
- **细节精致**：注重细节设计，提升整体品质感

## 🎨 视觉设计原则

### 1. 配色系统

#### 1.1 科技感配色方案
```
主色调：
- 主色（Primary）：#00FF88（霓虹绿）
- 主色深（Primary Dark）：#00CC6A
- 主色发光（Primary Glow）：#00FF88

辅助色：
- 橙色（Accent Orange）：#FF6B35（提醒、警告、徽章）
- 蓝色（Accent Blue）：#4CC9F0（链接、次要操作）
- 紫色（Accent Purple）：#B388FF（特殊装饰）

背景色：
- 深色背景（Bg）：#0A0E27
- 浅色背景（Bg Light）：#141829
- 卡片背景（Bg Card）：rgba(255, 255, 255, 0.05)
- 毛玻璃（Bg Glass）：rgba(255, 255, 255, 0.08)

文字色：
- 主文字（Text Primary）：#FFFFFF
- 次文字（Text Secondary）：rgba(255, 255, 255, 0.7)
- 弱文字（Text Weak）：rgba(255, 255, 255, 0.5)
- 禁用文字（Text Disable）：rgba(255, 255, 255, 0.3)

边框色：
- 边框（Border）：rgba(255, 255, 255, 0.1)
```

#### 1.2 配色使用规范
- **主色应用**：重要按钮、强调文字、进度条、激活状态
- **辅助色应用**：次要操作、提醒信息、特殊标识
- **背景色应用**：深色背景营造科技感，半透明卡片增加层次
- **文字色应用**：确保对比度≥4.5:1，保证可读性

### 2. 字体系统

#### 2.1 字体层级
```
H1 - 大标题：28px / 700 / 1.2
  - 页面主标题、重要数据展示

H2 - 中标题：20px / 600 / 1.3
  - 卡片标题、模块标题

H3 - 小标题：18px / 600 / 1.4
  - 分组标题、列表标题

Body - 正文：16px / 400 / 1.5
  - 正文内容、描述文字

Caption - 辅助：14px / 400 / 1.4
  - 辅助说明、时间、标签

Small - 极小：12px / 400 / 1.5
  - 极小文字、标签、时间戳
```

#### 2.2 字体规范
- **字体族**：`-apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", sans-serif`
- **数字字体**：使用 `font-variant-numeric: tabular-nums` 保证数字对齐
- **字重使用**：标题使用 600-700，正文使用 400，强调使用 700-800
- **字间距**：标题使用 `letter-spacing: 0.5px`，增加现代感

### 3. 间距系统

#### 3.1 间距规范
```
XS: 4px  - 图标与文字间距、微小间距
S:  8px  - 元素内部间距、小间距
M:  16px - 卡片内边距、标准元素间距
L:  24px - 模块间距、大元素间距
XL: 32px - 页面边距、大模块间距
XXL: 48px - 页面顶部/底部大间距
```

#### 3.2 间距使用原则
- **一致性**：相同类型的元素使用相同的间距
- **呼吸感**：避免元素过于紧密，保持适当留白
- **层次感**：通过间距大小区分信息层级

### 4. 圆角系统

#### 4.1 圆角规范
```
卡片：20px - 主要内容卡片（科技感圆角）
按钮：16px - 主要按钮
小按钮：12px - 次要按钮、标签
输入框：8px - 文本输入框
头像：50% - 圆形头像
图标容器：8px - 图标背景
```

#### 4.2 圆角使用原则
- **现代感**：使用较大的圆角值，营造现代、友好的感觉
- **一致性**：相同类型的元素使用相同的圆角值
- **渐变过渡**：圆角与阴影、边框配合使用

### 5. 阴影系统

#### 5.1 阴影规范
```
卡片阴影：
  0 8px 32px rgba(0, 0, 0, 0.3)
  - 普通内容卡片

按钮发光阴影：
  0 0 20px rgba(0, 255, 136, 0.4),
  0 8px 24px rgba(0, 255, 136, 0.2)
  - 主要按钮的霓虹发光效果

悬浮阴影：
  0 12px 40px rgba(0, 0, 0, 0.5),
  0 0 60px rgba(0, 255, 136, 0.3)
  - 悬浮、hover状态的增强阴影

内阴影：
  inset 0 2px 4px rgba(0, 0, 0, 0.2)
  - 输入框、进度条的内部阴影
```

#### 5.2 阴影使用原则
- **层次感**：通过阴影营造空间层次
- **发光效果**：重要元素使用霓虹发光阴影
- **适度使用**：避免过度使用阴影，保持视觉清晰

### 6. 毛玻璃效果

#### 6.1 毛玻璃规范

## 💫 动画设计原则

### 1. 动画时长规范
```
即时反馈：100-150ms
  - 按钮点击、状态切换

快速动画：200-300ms
  - 页面切换、弹窗出现

标准动画：400-600ms
  - 元素进入、退出动画

慢速动画：800ms-1.5s
  - 复杂动画、庆祝动画

循环动画：2s-3s
  - 呼吸动画、扫描动画
```

### 2. 缓动函数
```
ease-in-out：标准过渡（默认）
  - 页面切换、弹窗动画

ease-out：快速进入，慢速结束
  - 元素进入动画、成功反馈

ease-in：慢速开始，快速结束
  - 元素退出动画

cubic-bezier(0.5, 0, 0.5, 1)：自定义曲线
  - 特殊动画效果
```

### 3. 核心动画效果

#### 3.1 呼吸动画
**用途**：吸引用户注意的重要按钮
```css
@keyframes breathing {
  0%, 100% {
    transform: scale(1);
    box-shadow: var(--shadow-button);
  }
  50% {
    transform: scale(1.02);
    box-shadow: 增强发光阴影;
  }
}
```
**时长**：2s  
**循环**：infinite  
**缓动**：ease-in-out

#### 3.2 光泽扫描动画
**用途**：卡片、按钮的科技感装饰
```css
@keyframes shine {
  0% {
    transform: rotate(0deg) translate(-50%, -50%);
  }
  100% {
    transform: rotate(360deg) translate(-50%, -50%);
  }
}
```
**时长**：3s  
**循环**：infinite

#### 3.3 脉冲动画
**用途**：图标、重要元素的强调
```css
@keyframes pulse {
  0%, 100% {
    transform: scale(1);
    opacity: 1;
  }
  50% {
    transform: scale(1.05);
    opacity: 0.9;
  }
}
```
**时长**：2s  
**循环**：infinite

#### 3.4 庆祝动画
**用途**：打卡成功、成就达成
- **勾选动画**：SVG路径描边动画（600ms）
- **烟花动画**：粒子爆炸效果（1.5s）
- **弹窗出现**：淡入+上移（300ms）

#### 3.5 进度条动画
**用途**：进度条的光泽流动效果
```css
@keyframes progressShine {
  0% {
    transform: translateX(-100%);
  }
  100% {
    transform: translateX(200%);
  }
}
```

### 4. 动画使用原则
- **有意义**：每个动画都有明确的目的，不是装饰性的
- **适度使用**：避免过度动画，保持界面流畅
- **性能优先**：使用 `transform` 和 `opacity` 而非 `width`、`height`
- **尊重用户**：提供减少动画的选项（系统偏好设置）

- **问候语**：根据时间段显示不同问候（早上好/下午好/晚上好）

## 📐 布局设计原则

### 1. 布局系统

#### 1.1 网格系统
- **列数**：主要使用 2-3 列布局
- **间距**：列间距统一为 16px
- **响应式**：适配不同屏幕尺寸

#### 1.2 布局方式
- **Flexbox**：用于一维布局（横向、纵向）
- **Grid**：用于二维布局（卡片网格）
- **Position**：用于绝对定位（悬浮按钮）

### 2. 内容层级

#### 2.1 视觉层级
- **重要性**：重要内容使用更大字号、更明显颜色
- **位置**：重要内容放在视觉中心或上方
- **对比**：通过颜色、大小、阴影营造层次

#### 2.2 信息架构
- **主次分明**：核心功能突出，次要功能弱化
- **分组清晰**：相关内容分组展示
- **留白合理**：适当的留白增强可读性

### 3. 响应式设计

#### 3.1 断点设置
- **小屏**：< 375px
- **标准**：375px - 414px
- **大屏**：> 414px

#### 3.2 适配策略
- **弹性布局**：使用 flex、grid 自适应
- **百分比宽度**：避免固定宽度
- **媒体查询**：必要时使用媒体查询

## 🔧 技术实现原则

### 1. 性能优化

#### 1.1 动画性能
- **使用 transform**：优先使用 `transform` 和 `opacity`
- **避免重排**：避免触发 layout 的属性
- **GPU加速**：使用 `will-change` 提示浏览器
- **减少动画**：避免同时运行过多动画

#### 1.2 渲染优化
- **虚拟滚动**：长列表使用虚拟滚动
