# 坚持打开小程序需求文档

> 版本：V1.0  
> 更新时间：2025-11-18  

---

## 1. 产品定位与目标

- 产品定位：面向需要培养自律习惯的用户，提供“50 天坚持计划”为核心的打卡类小程序，帮助用户规划、执行并复盘个人成长计划。
- 核心目标：
  1. 降低制定计划门槛，提供模板化推荐方案。
  2. 强化日历式打卡与可视化反馈，提升坚持率。
  3. 通过成就卡片、勋章、好友互动等正反馈，形成长期留存。

## 2. 用户画像与痛点

| 角色 | 特征 | 主要痛点 |
| --- | --- | --- |
| 自律初学者 | 18-30 岁学生/白领，缺乏经验 | 不知道如何制定可执行计划，容易放弃 |
| 轻量健身爱好者 | 关注体态和健康 | 需要每日提醒和进度可视化 |
| 自我管理达人 | 记录控、喜欢数据分析 | 需要精细化统计和多日历对比 |

痛点总结：难以坚持、缺乏提醒、反馈滞后、缺乏成就感、缺少模板。

## 3. 核心场景与流程

1. **创建计划流程**  
   首页启动 → 选择推荐计划/自定义 → 设置目标频次、起止时间、提醒 → 生成计划。
2. **每日打卡流程**  
   进入每日打卡页 → 查看今日任务 → 勾选完成 → 记录感受/上传图片 → 获得积分+连击展示。
3. **复盘与激励流程**  
   通过日历或计划详情查看累计天数、完成率 → 领取阶段勋章 → 分享成就卡片给好友。

## 4. 功能需求

### 4.1 首页（参考图 1）

- 展示用户头像、昵称、累计坚持天数、进行中的计划数量。
- “所有坚持计划”模块：列出最近的 3 个计划卡片（名称、剩余天数、进度条、今日提醒状态）。
- 快捷入口：`创建计划`、`健康打卡`、`成长记录` 等按钮。
- 底部 Tab：`首页` `计划` `日历` `我的`。
- Banner 区用于活动、任务或系统公告（如“50 天重启人生”）。

### 4.2 创建计划（参考图 2）

- 入口：首页 + 计划列表页 + 推荐计划页。
- 表单字段：
  - 计划名称（默认 XX 天坚持 XX）
  - 每日时长或次数限制（如 30 分钟、1 次）
  - 执行频次（每日、每周 5 天、自定义）
  - 计划周期：开始日期、结束日期（默认 50 天，可调 7-100 天）
  - 提醒方式：小程序消息 / 微信订阅消息 / 无提醒
  - 提醒时间（多选）
  - 激励语：系统推荐 + 用户自填
- 提供“保存为模板”开关。
- 校验：开始日期 ≥ 今天，结束日期 ≥ 开始日期；名称≤20字；提醒时间不得重复。

### 4.3 推荐计划（参考图 3）

- 列表展示官方计划：配图、简介、适用人群、标签、计划周期。
- 每个推荐计划提供：`立即参加`、`收藏`、`详情`。
- 推荐算法（规则版）：按热度、最新、标签过滤。
- 官方计划示例：
  1. “1 小时效率工作流”  
  2. “1 小时力量训练”  
  3. “健康 10 打卡” 等。

### 4.4 计划详情（参考图 4、5）

- 头部显示：计划名称、进行天数/总天数、完成次数、完成率。
- 关键指标：坚持天数、当前进度、剩余天数、打卡率。
- 计划设置：开始结束日期、每日目标、提醒设置、是否共享。
- 行动按钮：`今日打卡`、`补打卡`、`查看勋章`、`暂停计划`、`删除计划`。
- 数据可视化：
  - 折线 + 柱状混合图显示每周/每日完成情况。
  - 日历热力视图：完成日高亮、未完成灰色、补卡黄色。
  - “Day X/50”卡片显示连续天数。
- 文案激励区：显示系统鼓励语，可自定义。

### 4.5 日历与打卡页（参考图 6）

- 月历视图：显示当月每日打卡状态。支持左右切换月份。
- 页脚统计：本月坚持天数、连续天数、完成率。
- “今日打卡”模块：
  - 当日任务列表（支持多计划同日）
  - 打卡方式：单击完成 + 上传图片/文字/心情（可选）
  - 支持“补卡”逻辑（需消耗补卡券或积分）。
- 数据同步：打卡后自动更新计划详情、勋章和首页统计。

### 4.6 50 天计划进度（参考图 7）

- 显示多个计划的进度滚动表：`Day X/50`。
- 统计信息：累计完成天数、完成率、剩余天数。
- 支持按计划切换查看细节。
- 提供“健康教练”小助手提示，基于数据给出建议。

### 4.7 个人中心（参考图 8 左）

- 用户头像、昵称、加入天数、等级。
- 入口：`我的勋章`、`健康守护`、`健身教练`、`积分商城`、`关于产品`、`用户协议`。
- 日程自动同步：允许用户选择同步到系统日历。
- 支持切换皮肤/主题。

### 4.8 成就卡片（参考图 8 右）

- 当用户达成阶段成就（如坚持 7/21/50 天）时弹窗展示。
- 卡片内容：昵称、计划名称、完成天数、勋章图案、励志文案、完成时间。
- 支持保存到相册、分享到微信群。
- 可租用“应援券”或“鼓励信”作为社交货币。

### 4.9 系统与通知

- 订阅消息模板：
  1. 每日计划提醒（含计划名、任务摘要、打卡入口）。
  2. 连续断签预警（当连续缺席 ≥2 天）。
  3. 成就解锁通知。
- 待办提醒：进入小程序后弹出今日待办列表。
- 运营位推送：Banner + 弹窗 + 消息中心。

## 5. 数据指标与运营策略

| 指标 | 计算逻辑 | 目标 |
| --- | --- | --- |
| DAU | 当日访问用户数 | 5k（上线 3 个月） |
| 新增计划数 | 当日创建的计划总数 | ≥500 |
| 7 日留存 | 7 日内至少打卡 4 次/计划仍在进行 | ≥40% |
| 平均打卡完成率 | 完成次数 / 计划总目标次数 | ≥60% |
| 分享率 | 产生成就分享/完成打卡分享的用户占比 | ≥20% |

运营策略：
- 每周四上线新计划模板 + 主视觉。
- 设置挑战赛榜单，按坚持天数排名并发放虚拟勋章。
- 结合节日推活动（如“春季焕新 30 天”）。

## 6. 非功能需求

- **性能**：页面首次打开 ≤ 1.5s；关键操作响应 ≤ 0.5s。
- **可靠性**：打卡数据需本地缓存 + 云端冗余，避免丢失。
- **安全**：用户数据加密存储，遵守微信小程序隐私合规。
- **兼容性**：支持微信 8.0+；适配 iOS/Android 主流 720-1440 宽度。
- **可运营性**：关键文案、活动位通过 CMS 配置。

## 8. 技术架构

### 8.1 技术栈

#### 核心框架
- **UniApp** - 跨平台开发框架
- **Vue 3** - 前端框架（使用 Composition API）
- **TypeScript** - 类型系统
- **微信云开发（CloudBase）** - 后端运行环境、云函数、数据库、云存储

#### 状态管理
- **Pinia** (替代 Vuex) - Vue 3 推荐的状态管理库
- 或 **Vuex 4.x** - 兼容 Vue 3 的版本

#### UI 组件库
- **uView Plus** (Vue 3 版本) 或 **uView UI 3.x**
- 如无 Vue 3 版本，考虑 **uni-ui** 或其他 Vue 3 兼容组件库

#### 构建工具与配置
- **Vue CLI 5.x** 或 **Vite** (推荐，Vue 3 官方推荐)
- **Webpack** / **Vite** - 打包工具
- **Babel** - JavaScript 编译器
- **PostCSS** - CSS 后处理器
- **Sass/SCSS** - CSS 预处理器

#### 工具库
- **lodash** - 工具函数库
- **dayjs** - 日期处理库（替代 moment.js）
- **core-js** - JavaScript 标准库 polyfill
